# Умный поиск источников

Приложение `apps.citations` — отдельный контур RAG для поиска публикаций, которыми
можно подтвердить конкретные утверждения загруженной статьи.

## Как работает

1. Индексатор читает `journal_articles_full_metadata.csv`, HTML-карточки и PDF из
   `downloads_elibrary`.
2. PDF разбиваются на самостоятельные смысловые фрагменты. Заголовок, секция,
   ключевые слова и метаданные индексируются вместе с текстом.
3. Анализатор выделяет утверждения без ссылок и классифицирует потребность:
   `topic`, `method`, `data`, `task` или `result`.
4. Для утверждения строятся русскоязычный и англоязычный запросы.
5. Retrieval объединяет SQLite FTS5 и векторную близость. Если настроен локальный
   `/v1/embeddings`, используются его эмбеддинги; иначе работает полностью
   локальный hashing-вектор.
6. Настроенная локальная LLM проверяет найденные фрагменты, отбрасывает простые
   тематические совпадения и объясняет рекомендацию.
7. Пользователь видит авторов, год, DOI/EDN, процент соответствия, объяснение и
   точный связанный фрагмент. Выбранные источники можно скопировать или вставить
   в DOCX вместе со списком литературы.

## Подготовка индекса

```powershell
python manage.py build_citation_index
```

Индекс хранится в `tmp/citation_index.sqlite3`, не попадает в Git и безопасно
пересоздаётся атомарной заменой. На сервере команду следует выполнять после
обновления корпуса. При отсутствии индекса первый поиск может собрать его
автоматически (`CITATION_INDEX_AUTO_BUILD=1`).

## Локальные модели

Для анализа утверждений и реранкинга используются общие настройки
`AI_PROVIDER=openai_compatible`, `AI_BASE_URL` и `AI_MODEL`.

Для настоящих dense embeddings задаются:

```dotenv
CITATION_EMBEDDING_BASE_URL=http://127.0.0.1:8080/v1
CITATION_EMBEDDING_MODEL=bge-m3
CITATION_EMBEDDING_API_KEY=
```

После смены embedding-модели индекс нужно перестроить.
